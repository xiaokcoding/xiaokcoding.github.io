解决 Windows 本地端口被系统“禁止绑定”问题（如 bind: access denied）

发表于2026-01-15|更新于2026-01-15
|总字数:234|阅读时长:1分钟|浏览量:

现象：
程序（如 V2Ray、Clash、自建 HTTP 服务）无法监听 108xx、20000 等端口，报错：

原因：
Windows 启用 Hyper-V / WSL2 / 沙盒后，会自动预留大量本地 TCP 端口（通过 HNS 服务），即使端口未被占用，普通程序也无法绑定。

解决方案

管理员权限运行，执行后必须重启电脑才能生效。

验证结果

重启后运行：

  • 若输出中不再包含 10000~30000 等常用代理端口段，说明已修复。
  • 仅剩 5357 等系统固定保留端口（可忽略）。
