chsrc：win11无法更换python源

发表于2025-12-31|更新于2025-12-31
|浏览量:

Issues：win11无法更换python源

问题分析

从日志来看，chsrc 在检测到 python3 命令后直接报错，提示该命令是“微软商店的占位符”。

但从 Get-Command python -All 的输出可以看到，系统中实际上存在两个 python.exe：

  • 一个版本号为 3.13.11，路径位于 AppData\Local\Programs
  • 另一个版本号为 0.0.0.0，路径位于 AppData\Local\Microsoft\WindowsApps

这说明当前 Windows 环境中，python 命令并不是一个“标准可控”的 Python 解释器。Windows 10 / 11 默认启用了 App Execution Aliases（应用执行别名） 功能。

即使用户没有手动安装 Python，系统中也会存在：

  • python.exe
  • python3.exe

这些命令实际上只是 Microsoft Store 的跳转占位符，并不是真正可用的 Python 解释器。

而当用户通过 Microsoft Store 安装 Python 后：

  • Python 可执行文件位于受限目录（WindowsApps）
  • 路径不可枚举
  • pip / 配置文件位置不可控

因此，像 chsrc 这类需要修改 pip 配置的工具，会明确拒绝该环境。

解决方案

Step 1：卸载 Microsoft Store Python

首先，在 设置 → 应用 → 已安装的应用 中卸载：

  • Python 3.13.x (64-bit)
  • Python Launcher

Step 2：关闭 Python 执行别名

进入 设置 → 应用 → 高级应用设置 →应用执行别名，关闭：

  • python.exe
  • python3.exe

Step 3：安装标准 Python（推荐 Scoop）

使用 Scoop 安装标准版 Python：

scoop install python

Step 4：验证 Python 来源

Get-Command python -All

确保路径指向 Scoop 目录，而不是 WindowsApps。

Step 5：重新执行 chsrc

chsrc set python，此时可以正常完成 pip 源的切换。
