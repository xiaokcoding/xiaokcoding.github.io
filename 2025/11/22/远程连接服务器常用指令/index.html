1. ssh-keygen — 生成 SSH 密钥对

功能

生成用于公钥认证的密钥对（私钥 + 公钥）。

基本语法

常用参数

参数 说明
-t type 密钥类型：ed25519（推荐）、rsa、ecdsa
-b bits 密钥长度（仅 RSA 有效，建议 ≥4096）
-C comment 注释（通常为邮箱）
-f filename 指定保存路径
-N passphrase 设置密钥密码（空值表示无密码）
-R hostname 从 known_hosts 中删除指定主机的旧公钥记录（不写入新密钥，仅清理）

示例

文件说明

  • 私钥：id_ed25519（权限应为 600）
  • 公钥：id_ed25519.pub（可公开）

2. ssh-copy-id — 自动部署公钥

功能

将本地公钥自动追加到远程主机的 ~/.ssh/authorized_keys，实现免密登录。

基本语法

常用参数

参数 说明
-i identity_file 指定 公钥文件（必须是 .pub）
-p port 指定 SSH 端口
-n 模拟运行（不实际写入）
-f 强制复制（即使已存在）

示例

3. SSH 配置文件 ~/.ssh/config

功能

通过别名简化连接命令，支持自定义端口、用户、密钥、保活、超时等，提升效率与稳定性。

示例配置（通用模板）

当然可以！以下是对你的 ~/.ssh/config 配置段落的逐行注释说明，保持简洁清晰，便于理解每项配置的作用：

常用配置项

配置项 说明
HostName 实际 IP 或域名
User 登录用户名
Port SSH 端口（默认 22）
IdentityFile 指定私钥路径
IdentitiesOnly yes 只使用指定密钥，避免尝试其他密钥
ServerAliveInterval N 每 N 秒发送一次保活包，防止连接因闲置断开
ServerAliveCountMax N 最多允许 N 次保活无响应，之后断开连接
TCPKeepAlive yes 启用底层 TCP keepalive（辅助应用层保活）
ConnectTimeout N 连接阶段超时时间（秒）
